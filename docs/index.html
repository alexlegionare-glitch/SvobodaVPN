Как пользоваться

  1. Запусти SvobodaVPN-Setup.exe, нажми «Да» на запрос системы — обычный мастер установки.
  2. На рабочем столе появится значок «Свобода VPN». Открой его.
  3. Нажми «+ Добавить сервер», вставь свою ссылку и нажми большую кнопку. Зелёная надпись «Защищено» = всё работает.
💡 Свой сервер: жми «+ Добавить сервер» и вставь ссылку. В продвинутом режиме (ссылка-переключатель внизу окна) появятся «Серверы» со всеми настройками и «Подписка» с авто-обновлением.

Не подключается? 3 шага

  1. Нажми кнопку ещё раз. Блокировка ставится только в момент соединения, и больше половины срабатываний — ложные. Переподключение часто оживляет канал (приложение уже пробует это само).
  2. Смени сервер или подними свой (раздел ниже). Режут в основном по IP / подсети сервера, а не по протоколу — свежий чистый VPS обходит блок лучше любых настроек.
  3. Продвинутый режим → «Серверы» → uTLS-отпечаток: переключи chrome ↔ firefox. Смена отпечатка иногда снимает блок.

Свой VPN с нуля — для новичка

Без знаний и без копания в настройках. Купил VPS → вставил одну команду → скопировал ссылку. ~10 минут, ~150–300 ₽/мес.
  1. Купи VPS за рубежом (ОС Ubuntu 22 или 24, доступ root). Дёшево: Aeza, Hetzner, 4VPS, PQ.Hosting. Запиши IP и пароль root.
  2. В панели хостера открой веб-консоль сервера (кнопка Console / VNC) или подключись по SSH.
  3. Вставь одну команду и нажми Enter:
    bash <(curl -fsSL https://raw.githubusercontent.com/alexlegionare-glitch/SvobodaVPN/main/setup.sh)
  4. Через минуту скрипт напишет готовую ссылку вида vless://…#Мой сервер — скопируй её.
  5. В приложении «Свобода ВПН» → Серверы → вставь ссылку → Добавить → выбери и подключись.
💡 Скрипт сам ставит сервер (sing-box), делает сертификат и настраивает обход ТСПУ (VLESS + TLS без SNI). Тебе остаётся только вставить ссылку в приложение.
Продвинутым: вручную / подписка / выдать ключ другим
  • Вручную: панель 3X-UI — bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh), создать VLESS+Reality, скопировать ссылку.
  • Выдать ключ другому: добавь нового клиента (UUID) в инбаунд, отдай его персональную ссылку.
  • Подписка с ротацией (как у платных): отдавай файл-список ссылок по HTTPS, URL вставляется в «Подписка» — мёртвые узлы меняются на живые.
